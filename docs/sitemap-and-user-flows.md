# 페이지맵 & 유저 플로우

**요약:** 개인 블로그의 주요 페이지 구조와 핵심 유저 플로우를 정리한 문서입니다.

---

## 페이지맵

- **퍼블릭 (비인증 사용자 접근)**
  - 홈: `/` — 최신/추천 포스트 노출
  - 포스트 목록: `/posts` — 페이징/필터
  - 포스트 상세: `/posts/[id]`
  - 검색: `/search` (SearchBar 연동)
  - 태그별 목록: `/tags/[tag]`
  - 약력/소개: `/about`
  - 연락처: `/contact`
  - 404 페이지

- **인증 관련**
  - 로그인: `/auth/login`
  - 회원가입: `/auth/register`
  - 비밀번호 재설정: `/auth/reset`

- **인증 사용자(작성자)**
  - 마이 페이지: `/me` — 최근 활동, 프로필 요약
  - 내 포스트 목록: `/me/posts`
  - 새 포스트: `/posts/new`
  - 포스트 편집: `/posts/[id]/edit`
  - 프로필 설정: `/me/settings`

- **관리/운영(선택)**
  - 대시보드(Admin): `/admin` — 사용자/포스트 관리
  - 배포/로그/CI 상태 페이지

---

## 핵심 유저 플로우

- **방문자(읽기 전용)**
  1. 진입: 홈 또는 검색 결과 → 목록 확인
  2. 포스트 선택 → 상세 페이지 열람
  3. 공유 또는 태그 클릭 → 관련 포스트 이동
  - 성공 기준: 포스트 열람 완료, 체류 시간 측정

- **독자(검색 → 읽기)**
  1. 검색어 입력 (`/search`) → 결과 목록
  2. 필터/정렬 적용 → 원하는 글 선택
 3. 글 읽음 → 필요시 회원가입 유도(댓글/좋아요)

- **인증된 작성자(글 작성)**
  1. 로그인(`/auth/login`) 또는 세션 유지
  2. `New Post` 진입 → 제목/본문 작성
  3. 이미지 업로드 (Supabase Storage) → 본문에 삽입
  4. 저장 또는 공개(Publish) → 리다이렉트 `/posts/[id]`
  - 필요 API: Supabase Auth, Posts API, Storage

- **포스트 수정 / 관리**
  1. `/me/posts`에서 내 글 선택
  2. 편집(`/posts/[id]/edit`) → 변경 저장
  3. 버전 또는 임시저장(초안) 지원 권장

- **댓글 쓰기(인증 필요)**
  1. 댓글 버튼 클릭 → 인증 여부 확인
  2. 미인증: `/auth/login`로 리다이렉트
  3. 인증 후 댓글 작성 → API 호출 → UI 반영

- **프로필 편집(인증된 유저)**
  1. `/me/settings` 진입
  2. 프로필 이미지(또는 아바타) 업로드 → Storage 연동
  3. 변경 저장 → 프로필 업데이트

---

## 구현 메모(우선순위/기술 매핑)

- 인증: Supabase Auth (Email) — 로그인/회원가입/세션
- 이미지: Supabase Storage — 업로드/URL 저장
- 포스트 저장: `lib/posts.ts` 기반 API (CRUD)
- 검색/필터: 클라이언트-사이드 필터 + 서버 검색 엑스포즈
- UI 규칙: Server Components 기본, 클라이언트 필요 시 `"use client"`

---

원하시면 각 유저 플로우를 시퀀스 다이어그램(mermaid)로 변환하거나, 우선순위별 작업 분해(작업 항목 + 예상 소요)까지 만들어 드리겠습니다.
